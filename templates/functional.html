Functional

Functional programming is a pure language. This means that you cannot do anything that has a side effect. For example, if you evaluate an expression, you can evaluate it as many times as you want with the same arguments and it will always return the exact same value as the first time you evaluated. Essentially means that you cannot evaluate an expression that will later cause evaluating that expression to result in a different value.

Languages: Racket, Haskell, Charity, Elm, Miranda, D, Scheme and more!
