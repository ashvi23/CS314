Imperative

Imperative languages involves a sequence of statements. These sequence of steps determines how to reach a certain goal or changes the state of the program. Imperative explicitly tells a computer how to accomplish a goal by performing the task line by line, or step by step. Typically, imperative programs are grouped into code blocks that is easier to follow. The main focus of imperative languages is to how to achieve the goal at hand. Interestingly, imperative language is based on the Van Neumann architecture. This means that imperative langauges have a close relation to machine architecture. Unlike functional programming, imperative languages allows side effects. This means that the program state can change and reassignment is legal. Thus, functions can produce varying different outputs for the same function. Imperative languages is advantagous over functional programming langauges in that it is much easier to learn. It allows loops which are much more intuitive than recursion.
