Static

Static languages are languages where variables are bounded by both a type and to an object. It is optional to bind to an object in static languages. If it is not, it will be null. When a variable is assigned to a type, it can only be bounded to objects that fall under that same type during execution. It will never be bounded to an object that has a different type. For static languages, types are always checked before run time. Compiling beforehand is one of the key features that differs it from dynamic programming. The advantage is that type errors are often time thrown before run time, avoiding some possible unexpected errors during run time. Since static typed languages compile before run time, static languages are often times have better performance during run time than dynamic languages.
